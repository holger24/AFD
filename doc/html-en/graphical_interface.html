The Graphical User Interface

The graphical user interface of the AFD is based on the X Window System from MIT. The X Window System is a network transparent window system which runs on a wide range of computing and graphics machines. For more information read the documents that came with your X Window System.

Currently the AFD has the following windows:

afd_ctrl The main window to control and monitor the AFD. With this window you can call all the other windows of the AFD.
dir_ctrl The main window to control and monitor the input part of AFD.
show_log Shows the contents of the System-, Receive-, Transfer- and Transfer Debug Log files.
edit_hc Change configuration for the host name.
afd_info Information window showing more details about the remote host.
show_ilog Shows the contents of the input log file.
show_olog Shows the contents of the output log file. With this window it is possible to resend files that have been archived.
show_dlog Shows the contents of the delete log file.
show_queue Shows the contents of the input and/or output queue. With this window it is possible to delete files from the queue or send from the queue to some other destination.
xsend_file Send file to some other destination. This is used by the show_olog and show_queue dialog.
mon_ctrl Dialog to monitor and control other AFD's.

All the above windows are separate windows, ie. you can start them without the afd_ctrl or afd_mon window.
